> [!abstract] 核心问题
> Claude Code 会记录用户输入的内容吗？辱骂、敏感词、代码内容会被上报吗？
> 答案：**用户输入内容默认被遮盖（REDACTED），analytics 系统从类型层面就禁止了字符串上报。**

## 用户输入默认不被记录

在 `src/utils/telemetry/events.ts:17`，有一个关键函数：

```typescript
function isUserPromptLoggingEnabled() {
  return isEnvTruthy(process.env.OTEL_LOG_USER_PROMPTS)
}

export function redactIfDisabled(content: string): string {
  return isUserPromptLoggingEnabled() ? content : '<REDACTED>'
}
```

所有用户输入在传给遥测系统之前，都会经过 `redactIfDisabled` 处理。**默认情况下，内容变成字面量 `<REDACTED>`**，只有用户主动设置环境变量 `OTEL_LOG_USER_PROMPTS=1` 才会开启内容记录（仅用于企业/内部调试场景）。

> [!important] 实际意义
> 无论你输入了什么——辱骂、密码、代码片段、文件路径——只要没有主动开启该环境变量，Anthropic 的 analytics 系统收到的永远是 `<REDACTED>`，而非真实内容。

## 类型系统作为隐私护栏

在 `src/services/analytics/index.ts:61`，analytics 事件的 metadata 类型被严格限制：

```typescript
type LogEventMetadata = { [key: string]: boolean | number | undefined }
```

**字符串类型被刻意排除在外。** 这意味着开发者从代码层面就无法把文本内容写进 analytics 事件——类型检查在编译时就会阻止这种行为。

这是"用类型系统实现隐私保护"的典型案例，比运行时 if 检查更可靠。

> [!tip] 设计启示
> 隐私保护不只是运行时的 if 判断，更应该体现在**类型设计**上。当某个字段类型只允许数字和布尔值时，开发者根本没有机会"不小心"记录了用户文字。

## 强制审查的特殊类型

在 `src/services/analytics/metadata.ts:57`，有一个命名极其特殊的类型：

```typescript
export type AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS = never
```

这个类型的值是 `never`，永远不能直接赋值，必须用 **类型断言（as）** 来使用：

```typescript
return toolName as AnalyticsMetadata_I_VERIFIED_THIS_IS_NOT_CODE_OR_FILEPATHS
```

这个设计的意图是：**每次有字符串需要进入 analytics，开发者必须显式写下这段"声明"，承诺"我已确认这不是代码或文件路径"。** 这是一种文档化的强制审查机制。

> [!example] 类比
> 就像医院要求护士在注射危险药物前必须大声朗读药名一样——不是技术上不能跳过，而是通过流程设计让"偷懒绕过"变得更难。

## MCP 工具名称也会被脱敏

在 `src/services/analytics/metadata.ts:70`：

```typescript
export function sanitizeToolNameForAnalytics(toolName: string): ... {
  if (toolName.startsWith('mcp__')) {
    return 'mcp_tool'  // 用户自定义的 MCP server 名称被替换
  }
  return toolName  // 内置工具名（Bash、Read、Write）保留
}
```

MCP 工具名格式为 `mcp__<server名>__<工具名>`，其中 server 名可能包含用户隐私（如公司内部系统名称）。因此统一替换为 `mcp_tool` 上报。

只有以下情况例外，可以保留原始名称：
- 官方 MCP 注册表中的工具（公开信息）
- claude.ai 代理的连接器
- 企业 local-agent 模式

## 实际上报的数据清单

| 上报 ✅ | 不上报 ❌ |
|--------|---------|
| 使用了哪个工具（类型名） | 用户输入的具体文字 |
| 工具调用成功/失败 | 代码内容、文件路径 |
| 模型名称、版本号 | MCP server 自定义名称 |
| 平台、操作系统 | 会话对话内容 |
| CPU/内存性能指标 | 任何字符串形式的用户数据 |
| 匿名 UUID（session/user） | 账号邮箱、真实身份 |

## 内容安全拦截在哪里发生？

Claude Code **客户端不做词汇过滤**，辱骂/有害内容的处理在 **Anthropic API 后端（Claude 模型层）** 进行。

`src/constants/cyberRiskInstruction.ts` 中只有一段系统提示词，定义了 Claude 对安全敏感请求的行为准则——这是**模型行为引导**，不是客户端词汇表过滤器。

> [!tip] 设计启示
> 内容安全 = 客户端隐私保护（不记录）+ 模型层行为引导（不执行有害操作）。两层各司其职，客户端不需要维护敏感词库。

## 如何关闭 Analytics？

用户可以通过以下方式退出数据收集：

```bash
# 在 settings.json 中设置
"disableTelemetry": true

# 或通过环境变量
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1
```

源码中 `isAnalyticsDisabled()` 函数会检查这些配置，完全停止所有上报。
