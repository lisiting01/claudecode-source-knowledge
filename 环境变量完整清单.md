# 环境变量完整清单

> [!abstract] 这是什么
> 这是 Claude Code 所有环境变量的速查表。按功能分类，标注了每个变量的用途和安全等级。当你想知道"某个变量是干什么的"或者"我该设哪个变量"的时候，来这里查。

## 怎么看这张表

- **安全等级**列说明这个变量能否从项目配置文件中设置：
  - ✅ **安全** = 在白名单内，任何来源都可以设置
  - ⚠️ **危险** = 不在白名单内，只有受信任来源（用户设置、CLI 参数、企业策略）可以设置
  - 🔒 **被清洗** = 在 GitHub Actions 的子进程中会被主动删除
- 安全等级的具体含义见 [[环境变量的安全过滤机制]]

## 一、认证与 API 路由

这一类变量决定了"请求发给谁"和"用什么身份"。

### 提供商选择

| 变量 | 用途 | 安全等级 |
|------|------|---------|
| `CLAUDE_CODE_USE_BEDROCK` | 启用 AWS Bedrock 作为 API 提供商 | ✅ 安全 |
| `CLAUDE_CODE_USE_VERTEX` | 启用 Google Vertex AI 作为 API 提供商 | ✅ 安全 |
| `CLAUDE_CODE_USE_FOUNDRY` | 启用 Anthropic Foundry 作为 API 提供商 | ✅ 安全 |
| `CLAUDE_CODE_SKIP_BEDROCK_AUTH` | 跳过 Bedrock 认证（测试用） | ✅ 安全 |
| `CLAUDE_CODE_SKIP_VERTEX_AUTH` | 跳过 Vertex 认证（测试用） | ✅ 安全 |
| `CLAUDE_CODE_SKIP_FOUNDRY_AUTH` | 跳过 Foundry 认证（测试用） | ✅ 安全 |

### API 密钥与认证

| 变量 | 用途 | 安全等级 |
|------|------|---------|
| `ANTHROPIC_API_KEY` | Anthropic API 密钥 | ⚠️ 危险 🔒 被清洗 |
| `ANTHROPIC_AUTH_TOKEN` | 替代认证令牌 | ⚠️ 危险 🔒 被清洗 |
| `CLAUDE_CODE_OAUTH_TOKEN` | OAuth 认证令牌 | ⚠️ 危险 🔒 被清洗 |
| `ANTHROPIC_FOUNDRY_API_KEY` | Foundry 专用 API 密钥 | ✅ 安全 🔒 被清洗 |
| `AWS_BEARER_TOKEN_BEDROCK` | Bedrock 认证令牌 | ⚠️ 危险 🔒 被清洗 |

> [!info] 为什么 FOUNDRY_API_KEY 是"安全"又"被清洗"？
> "安全"指的是可以从项目配置加载（不会把请求重定向到别的服务器）。"被清洗"指的是不传给子进程（防止被 prompt injection 窃取）。两者是不同维度的保护。

| `ANTHROPIC_BASE_URL` | 自定义 API 端点地址 | ⚠️ 危险 |
| `ANTHROPIC_BEDROCK_BASE_URL` | Bedrock 端点地址 | ⚠️ 危险 |

## 十一、Shell 设置中的危险命令

除了环境变量，`settings.json` 中还有一些**可执行 shell 命令**的设置项。它们的危险等级更高，因为可以直接运行任意代码：

| 设置项 | 用途 |
|--------|------|
| `apiKeyHelper` | 执行命令获取 API 密钥 |
| `awsAuthRefresh` | 执行命令刷新 AWS 认证 |
| `awsCredentialExport` | 执行命令导出 AWS 凭证 |
| `gcpAuthRefresh` | 执行命令刷新 GCP 认证 |
| `otelHeadersHelper` | 执行命令获取 OTEL 头部 |
| `statusLine` | 执行命令生成状态栏内容 |

这些设置项在源码中被标记为 `DANGEROUS_SHELL_SETTINGS`。
